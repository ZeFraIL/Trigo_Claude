## 🎯 Цель проекта
•	**Какую задачу решает приложение:**  
Помогает пользователям изучать тригонометрию с помощью интерактивных инструментов и визуализации.  
•	**Почему эта задача важна:**  
Тригонометрия часто вызывает сложности у студентов из-за абстрактности. Визуализация на тригонометрическом круге делает обучение наглядным и понятным.  
•	**Целевая аудитория:**  
Школьники, студенты и все, кто хочет освежить знания по тригонометрии.

## 📌 Требования к приложению
**Функциональные требования**  
•	Авторизация пользователя.  
•	Интерактивный тригонометрический круг с изменением угла.  
•	Отображение значений sin, cos, tan в реальном времени.  
•	Разделы теории, практики и тестов.  
•	Отслеживание прогресса обучения.  
**Нефункциональные требования**  
•	**Производительность:** Быстрая отрисовка графических элементов.  
•	**Удобство использования:** Интуитивно понятный интерфейс с вкладками.  
•	**Надёжность:** Стабильная работа при переходах между фрагментами.  
________________________________________
## 🧠 Общая архитектура
•	**Выбранный подход:**  
–	MVC (Model-View-Controller)  
•	**Почему выбран именно он:**  
Проект имеет среднюю сложность, и классическая архитектура Android (Activity/Fragment как контроллер) обеспечивает достаточную гибкость и простоту реализации без лишнего усложнения кода.  
•	**Основные компоненты системы:**  
–	Activity (MainActivity, LoginActivity) — управление жизненным циклом и навигацией.  
–	Fragments — логика отдельных экранов (Теория, Тесты и т.д.).  
–	Custom View (TrigonometryCircleView) — графическое отображение круга.

## 🧩 Подробное описание классов
**📌 Класс: MainActivity**  
**Роль:**  
Главный экран и координатор приложения.  
**Ответственность:**  
Управление фрагментами, проверка авторизации, обработка навигационного меню.  
**Основные методы:**  
- `onCreate()` — инициализация UI и проверка логина.  
- `loadFragment()` — динамическая замена фрагментов в контейнере.  
- `logout()` — сброс данных сессии.  
**Взаимодействие с другими классами:**  
Взаимодействует со всеми фрагментами и `LoginActivity`.  

**📌 Класс: TrigonometryCircleView**  
**Роль:**  
Пользовательский графический компонент.  
**Ответственность:**  
Отрисовка тригонометрического круга, векторов и проекций функций (sin, cos).  

**📌 Класс: TheoryFragment**  
**Роль:**  
Экран обучения.  
**Ответственность:**  
Управление интерактивным кругом и отображение математических данных.  
________________________________________
## 🔄 Диаграмма работы приложения
**Сценарий:**  
1. Запуск -> Проверка SharedPreferences -> Переход в LoginActivity (если не залогинен).  
2. LoginActivity -> Ввод данных -> Сохранение состояния -> MainActivity.  
3. MainActivity -> Выбор "Теория" -> Отображение TheoryFragment.  
4. Взаимодействие с SeekBar -> Обновление TrigonometryCircleView.

## 🎨 UI/UX анализ
•	**Почему интерфейс сделан именно так:**  
Выбрана кнопочная навигация снизу для быстрого доступа к основным разделам.  
•	**Какие принципы использованы:**  
–	**Простота:** Минимум лишних элементов на экране.  
–	**Логичность:** Последовательность "Теория -> Практика -> Тест".  
–	**Доступность:** Крупные элементы управления (SeekBar).  
•	**Что можно улучшить:**  
Добавить темную тему и поддержку альбомной ориентации для графиков.

## ⚙️ Работа с потоками
**Описание:**  
•	В основном используется **Main Thread**, так как вычисления тригонометрических функций не являются ресурсоемкими.  
•	**Как предотвращаются:**  
–	**Зависания (ANR):** Отсутствие тяжелых синхронных операций ввода-вывода.  
–	**Утечки памяти:** Использование `WeakReference` при необходимости и корректное управление жизненным циклом фрагментов.  
________________________________________
## 💾 Работа с данными
•	**Где хранятся данные:**  
SharedPreferences.  
•	**Почему выбран этот способ:**  
Для хранения статуса авторизации и простого прогресса пользователя это наиболее легкий и эффективный метод.  
•	**Как обеспечивается:**  
–	**Сохранность:** Автоматическая запись при изменении состояния.  
–	**Корректность:** Валидация данных перед записью.
