# Описание класса LoginActivity

## 1. Общая информация
*   **Имя класса:** `LoginActivity`
*   **Тип:** `Activity` (Экран авторизации)
*   **Назначение:** Отвечает за вход пользователя в систему и регистрацию новых пользователей. Сохраняет данные в памяти устройства.
*   **Взаимодействие:** После успешного входа передает управление `MainActivity`. Использует `SharedPreferences` для хранения учетных данных.

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `etUsername` | `EditText` | Поле ввода имени | `performLogin()`, `performRegistration()` |
| `etPassword` | `EditText` | Поле ввода пароля | `performLogin()`, `performRegistration()` |
| `btnLogin` | `Button` | Кнопка действия | `setupClickListeners()` |
| `isLoginMode` | `boolean` | Режим: вход или регистрация | `updateUI()`, `setupClickListeners()` |

## 3. Методы класса
**Имя метода:** `performLogin`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает:** Считывает введенные данные, сравнивает их с сохраненными в `SharedPreferences`. Если данные совпадают, вызывает `loginSuccess()`.
*   **Когда вызывается:** При нажатии кнопки "Войти".

**Имя метода:** `performRegistration`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает:** Проверяет корректность ввода и сохраняет нового пользователя (имя и пароль) в память.
*   **Когда вызывается:** При нажатии кнопки "Зарегистрироваться".

## 4. Жизненный цикл
*   **`onCreate()`:** Устанавливает внешний вид экрана и настраивает слушателей нажатий.

## 5. Взаимодействие с интерфейсом
*   **Элементы:** Поля ввода (`EditText`), кнопка (`Button`), текстовая ссылка для переключения режима.
*   **События:** Нажатие на кнопку выполняет проверку данных; нажатие на текст меняет режим (Вход <-> Регистрация).

## 6. Взаимодействие с другими компонентами
*   **Переходы:** При успехе запускает `MainActivity` и закрывает текущий экран (`finish()`).
*   **Данные:** `SharedPreferences.Editor` используется для записи данных пользователя.

## 7. Общая логика
Экран работает в двух режимах. В режиме "Вход" он ищет совпадение в памяти. В режиме "Регистрация" он записывает новые данные. Это простая имитация системы аккаунтов без использования интернета.

## 8. Объяснение простыми словами
`LoginActivity` — это **охранник на входе в клуб**. У него есть блокнот (`SharedPreferences`), где записаны имена и пароли гостей. Если вы уже в списке, он вас пропускает. Если нет — вы можете попросить его записать вас в блокнот (регистрация), и тогда он станет вас узнавать.
