# Описание класса MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** `Activity` (Экран приложения)
*   **Назначение:** Является основным экраном приложения, который управляет навигацией между различными разделами (Теория, Практика, Тесты и т.д.). Он отвечает за проверку того, вошел ли пользователь в систему, и за отображение соответствующего контента через фрагменты.
*   **Взаимодействие:** Взаимодействует с `LoginActivity` (для входа), `SharedPreferences` (для проверки сессии) и набором фрагментов (`TheoryFragment`, `PracticeFragment`, `QuizFragment` и др.).

## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `PREFS_NAME` | `String` | Имя файла настроек | `getSharedPreferences()` |
| `KEY_USER_LOGGED_IN` | `String` | Ключ для проверки входа | `isUserLoggedIn()`, `logout()` |
| `btnTheory`...`btnSettings` | `Button` | Кнопки навигации | `initializeViews()`, `setupClickListeners()` |
| `fragmentContainer` | `FrameLayout` | Контейнер для фрагментов | `loadFragment()` |
| `sharedPreferences` | `SharedPreferences` | Объект для работы с памятью | `onCreate()`, `isUserLoggedIn()` |

## 3. Методы класса
**Имя метода:** `onCreate`
*   **Тип метода:** `protected`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:** `Bundle savedInstanceState` (сохраненное состояние)
*   **Что делает:** Вызывается при создании экрана. Инициализирует настройки, проверяет, вошел ли пользователь (если нет — отправляет на `LoginActivity`), настраивает кнопки и загружает первый фрагмент (Теория).
*   **Когда вызывается:** Автоматически при запуске приложения.

**Имя метода:** `loadFragment`
*   **Тип метода:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `Fragment fragment` (объект фрагмента)
*   **Что делает:** Использует `FragmentManager` для замены текущего фрагмента в контейнере на новый. Это позволяет менять содержимое экрана без перезагрузки всей Activity.
*   **Когда вызывается:** При нажатии на кнопки навигации.

**Имя метода:** `logout`
*   **Тип метода:** `public`
*   **Возвращаемое значение:** `void`
*   **Что делает:** Сбрасывает флаг входа в `SharedPreferences` и перенаправляет пользователя на экран логина.
*   **Когда вызывается:** Может вызываться из фрагмента настроек.

## 4. Жизненный цикл
*   **`onCreate()`:** Инициализация всех компонентов. Здесь решается, какой экран показать пользователю первым.

## 5. Взаимодействие с интерфейсом
*   **Элементы:** Кнопки (`btn_theory`, `btn_practice` и т.д.) и контейнер (`fragment_container`).
*   **Связь с кодом:** Используется `findViewById`.
*   **События:** Обработка нажатий (`setOnClickListener`) для переключения между разделами.

## 6. Взаимодействие с другими компонентами
*   **Переходы:** Запуск `LoginActivity` через `Intent`.
*   **Данные:** Чтение состояния "залогинен" из `SharedPreferences`.

## 7. Общая логика
Класс работает как диспетчер. При запуске он проверяет "билет" (логин). Если всё в порядке, он показывает главное меню. При нажатии на любую кнопку меню, `MainActivity` убирает старый фрагмент и ставит на его место новый, соответствующий выбранному разделу.

## 8. Объяснение простыми словами
Представьте, что `MainActivity` — это **телевизор с пультом**. Сам телевизор (Activity) всегда один и тот же, но с помощью кнопок на пульте (кнопки навигации) вы переключаете каналы (фрагменты: мультики, новости, спорт). Если телевизор включил человек, который не оплатил подписку (не залогинился), телевизор сразу переключается на экран оплаты (`LoginActivity`).
